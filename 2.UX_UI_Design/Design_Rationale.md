# **สรุปเหตุผลการออกแบบ UX/UI \- BMI Calculator**

## **1\. การเลือกใช้สี (Color)**

# **สีหลัก: Indigo (\#6366f1)**

* # **สื่อถึงความน่าเชื่อถือทางการแพทย์**

* # **ไม่ก้าวร้าว เหมาะกับแอปสุขภาพ**

# **สีตามหมวดหมู่ BMI:**

* #  **น้ำเงิน (Underweight) \= ต้องการพลังงาน**

* # **เขียว (Normal) \= สุขภาพดี ปลอดภัย**

* #  **เหลือง (Overweight) \= สัญญาณเตือน**

* #  **แดง (Obese) \= ต้องให้ความสำคัญ**

#  **ใช้ Traffic Light System ที่คนเข้าใจง่าย**

## **3\. User Flow (ขั้นตอนการใช้งาน)** **เปิดแอป → เห็น Form ทันที**    **↓** **กรอกข้อมูล (ชื่อ, อายุ, เพศ, น้ำหนัก, ส่วนสูง)**    **↓** **กด Calculate BMI**    **↓** **เห็นผลทันที \+ BMI Scale \+ คำแนะนำ**    **↓** **บันทึกอัตโนมัติ → ดูกราฟแนวโน้ม**

# **หลักการ:**

* # **Zero Friction \= ใช้งานได้ทันทีไม่ต้อง Login**

* # **Instant Feedback \= คำนวณเร็ว ไม่ต้องรอ**

* # **Auto-save \= บันทึกอัตโนมัติไม่ต้องกดบันทึก**

## **4\. หลักการ UX ที่สำคัญ**

### **4.1 Fitts's Law**

* # **ปุ่ม Calculate ใหญ่ กดง่าย (Touch-friendly)**

* # **อยู่ตำแหน่งที่คาดหวัง (ด้านล่าง form)**

### **4.2 F-Pattern Reading**

* # **ข้อมูลสำคัญอยู่ซ้ายบน**

* # **ตาอ่านจากซ้ายไปขวา บนลงล่าง**

### **4.3 Visual Hierarchy**

* # **ตัวเลข BMI ใหญ่ที่สุด (ข้อมูลหลัก)**

* # **Category สีสะดุดตา**

* # **Health Tip รองลงมา**

# **4.4  Progressive EnhancementLevel 1: Form (ง่าย)** **Level 2: Result \+ สี** **Level 3: BMI Scale** **Level 4: คำแนะนำสุขภาพ**

# **Level 5: History \+ กราฟ (Advanced)** 

##  **5\. Data Visualization**

### **BMI Scale (แถบสี Gradient)**

* # **แสดงตำแหน่งของคุณบนสเกล 15-40**

* # **ใช้สีสื่อถึงระดับความเสี่ยง**

### **Line Chart (กราฟเส้น)**

* # **แสดงแนวโน้ม BMI ตามเวลา**

* # **มีเส้นอ้างอิง (18.5, 25, 30\) เปรียบเทียบได้**

#  **6\. Component Design**

### **Card-Based Layout**

* # **ใช้การ์ดแยกแต่ละ section (มาตรฐานสมัยใหม่)**

* # **มีเงา (shadow) สร้างความลึก 3 มิติ**

### **Form Design**

* # **Input ขนาดใหญ่ (เหมาะกับมือถือ)**

* # **เปลี่ยนสีขอบเมื่อ Focus**

* # **มี Placeholder บอกตัวอย่าง**

### **Button Hierarchy**

* # **Primary (Calculate) \= น้ำเงิน โดดเด่น**

* # **Secondary (Reset) \= เทา**

* # **Danger (Delete) \= แดง**

#  **7\. Accessibility (การเข้าถึง)**

#  **Color Contrast \= สีมีความต่างชัดพอ (WCAG AA)**    **Keyboard Navigation \= Tab ได้ตามลำดับ**    **Large Touch Targets \= ขนาดปุ่มใหญ่กว่า 44px**    **Semantic HTML \= ใช้ Tag ที่ถูกต้อง (h1, h2, button)**

##
